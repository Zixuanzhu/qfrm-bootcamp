Below is the agenda we follow to write and refine resumes, build a University web page and personal website, and learn some financial concepts while developing programming skills. Not necessarily in that order.

Agenda

  1. Overview of your online employment portfolio
    1. Your U of M home page.
    2. resume
    3. LinkedIn page
    4. personal website hosted with GitHub Pages. You'll need to create a free GitHub account. Use an appropriate name for your account, as it will be visible to potential employers.
  2. Install tools: GitHub client, Anaconda, Atom. We assume you have a word processor such as Google doc or MS Word.
    1. Create a GitHub account with a professional username.
    2. Download and install GitHub Desktop.
    3. Install Anaconda if you don't already have it. We prefer the 2.7 edition of Python, and recommend the graphical installer.
    4. Follow step one of the Conda 30 minute test drive
    5. Follow step four of the Conda 30 minute test drive.
    6. Launch Jupyter Notebook. You may want to create a special directory on your laptop for your Jupyter notebook files which you can maintain with GitHub.
  3. Getting started with GitHub
    1. From GitHub Desktop, click on the '+' in the upper left to create a new repository, create a GitHub repository named qfrm-python. For Windows, I suggest using the Local Path 'C:\Users\yourUserName\Documents\GitHub\'.
    2. Launch Atom (on Windows, you can launch it from GitHub Desktop via the icon in the upper right). Create a README.md file using Atom in your qfrm-python directory. Add this text to it: “These are programs I’ve written while learning Python. The implementation is in Jupyter notebooks."
    3. In GitHub Desktop, refresh the Changes tab in your qfrm-python project, and Publish. Then commit your changes (set Summary to something like "initial commit"), and sync.
    4. Log in to GitHub in your browser, and navigate to your qfrm-python repo. You should see your files listed, and your README.md file has been rendered as HTML.
  4. Getting started with Jupyter and Python
    1. Jupyter runs Python in your browser, using a Jupyter Notebook. The Notebook requires a local server to be running, which you will start using a shell script. Notebooks support Markdown, and python code with full access to everything you got from Anaconda. We will start by downloading a stubbed notebook called python-basics-notebook-empty.ipynb from python-templates. Click on the Raw button, select File->Save as..., chnage "Save as type:" to "All iles", and change the name to "python-basics-notebook.ipynb" before saving in your qfrm-python directory. NOTE: on Windows, you must explicitly save with
    2. Launch Anaconda2->Jupyter Notebook, and navigate to your qfrm-python directory.
    3. In GitHub Desktop, refresh the Changes tab in your qfrm-python project, commit your changes (set Summary to something like "added shell script"), and sync.
    4. Create a Jupyter notebook, and name it python-basics-notebook. We'll use it for the following...
    5. simple data types: int, float, boolean, string
    6. control structures: for loops, if/else
    7. collection data types: lists, dictionaries
    8. for loops with collections
    9. functions
    10. commit your work via GitHub Desktop, and sync.
  5. Python Basics
    1. From GitHub Desktop, create a GitHub repository , and name it python-basics-project.
    2. Create a README.md file using Atom in your python-basics-project directory. Add this text to it: “These are examples of programs I’ve written while learning Python. The implementation is in Jupyter notebooks."
    3. Create a Jupyter notebook, and name it python-basics-notebook. We'll use it for the following...
    4. simple data types: int, float, boolean, string
    5. control structures: for loops, if/else
    6. collection data types: lists, dictionaries
    7. for loops with collections
    8. functions
    9. commit your work via GitHub Desktop, and sync.
  6. Additional Python practice
    1. From GitHub Desktop, create a GitHub repository, and name it python-secret-project.
    2. Create a README.md file using Atom in your python-secret-project directory. Add this text to it: “These are programs I’ve written to practice Python. I am striving for concise, readable, and sufficiently efficient code."
    3. Create a Jupyter notebook, and name it python-secret-notebook. We'll use it for problems in Project Euler.
    4. commit your work via GitHub Desktop, and sync.
    5. Other practice problems you might consider...
      1. Google Code Jam
      2. Programming Praxis
  7. Python applied to interview questions
    1. From GitHub Desktop, create a GitHub repository , and name it python-interview-project.
    2. Create a README.md file using Atom in your python-finance-project directory. Add this text to it: “These are programs I’ve written to verify solutions to interview problems.""
    3. Solve problems 1.1, 3.1, 3.5, 3.8, 3.10, 3.11, 4.4, 5.10.
    4. commit your work via GitHub Desktop, and sync.
  8. Python in Finance
    1. From GitHub Desktop, create a GitHub repository , and name it python-finance-project.
    2. Create a README.md file using Atom in your python-finance-project directory. Add this text to it: “These are programs I’ve written to solve quantitative finance problems. I use the numpy and pandas libraries."
    3. Create a Jupyter notebook in your project, and name it zero-coupon...commit.
    4. Create a notebook named black-merton-scholes... and commit.
    5. Create a notebook named price-model... and commit.
    6. Create a notebook named historical-simulation... and commit.
    7. Create a notebook named brownian-motion... and commit.
    8. Create a notebook named monte-carlo-simulation... and commit.
    9. Create a notebook named risk-measures... and commit.
